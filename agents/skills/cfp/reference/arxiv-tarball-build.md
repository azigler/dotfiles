# arXiv tarball build (Shape B / paper submissions)

arXiv strongly prefers source over PDF. Their TeX Live 2025 system compiles
your sources directly. To work cleanly:

## What to ship

- **`main.tex`** — root LaTeX file
- **`main.bbl`** — pre-generated by BibTeX, filename matches main.tex
- **`references.bib`** — also include for archival completeness
- **`sections/*.tex`** — all section input files
- **`acmart.cls`** (or your venue's class file) — ship it to lock against
  TeX Live drift; arXiv uses your local copy when present
- **`*.bst`** (e.g. `ACM-Reference-Format.bst`) — bib style
- **Figures** as PDF (TikZ inline doesn't need separate files)

## What to EXCLUDE

- `.aux`, `.log`, `.toc`, `.lot`, `.lof`, `.dvi`, `.fdb_latexmk`, `.fls`, `.out`
- The compiled `.pdf` (arXiv generates its own)
- Hidden files / directories (anything starting with `.`)
- Journal templates, referee letters
- Files with embedded JavaScript
- Source code if not part of the paper itself

## Source audit before tarring

```bash
# Verify no \today (arXiv discourages)
grep -nE "\\\\today" main.tex sections/*.tex || echo OK

# Verify no xr package usage (won't work on arXiv)
grep -n "xr" main.tex || echo OK

# Verify no unescaped # / & in URLs in references.bib
grep -nE "url\s*=.*[#&]" references.bib || echo OK

# Verify .bbl is fresh
rm -f main.aux main.bbl main.blg main.fdb_latexmk main.fls main.log main.out
latexmk -pdf main.tex
ls -la main.bbl  # should be newer than references.bib
```

## Build the tarball

```bash
cd paper
mkdir -p /tmp/arxiv-build
cp main.tex main.bbl references.bib acmart.cls ACM-Reference-Format.bst /tmp/arxiv-build/
cp -r sections /tmp/arxiv-build/
[ -d figures ] && cp -r figures /tmp/arxiv-build/
cd /tmp/arxiv-build
tar czf ~/cfp/<slug>/arxiv-source.tar.gz .
ls -la ~/cfp/<slug>/arxiv-source.tar.gz
tar tzf ~/cfp/<slug>/arxiv-source.tar.gz
```

## Verify the tarball compiles cleanly

```bash
cd /tmp/arxiv-build
pdflatex -interaction=nonstopmode main.tex
pdflatex -interaction=nonstopmode main.tex
pdfinfo main.pdf | grep Pages
rm -f main.aux main.log main.out main.pdf  # clean up
```

If it compiles standalone (without invoking BibTeX, since `.bbl` is shipped),
arXiv's pipeline will succeed.

## Don't commit the tarball to the repo

It's reproducible from the source tree. Add to `.gitignore`:

```
arxiv-source.tar.gz
```

## arXiv submission notes

- **License:** CC-BY 4.0 to match most artifact bundles + open-access venues
- **Primary category:** likely `cs.SE` (Software Engineering); cross-list
  `cs.AI`, `cs.HC` as appropriate
- **Endorsement:** first-time submitters in `cs.SE` need an endorsement.
  Request from a colleague with 3+ papers in that category in the last
  5 years. 1–3 day async loop. NOT a proceedings blocker for most venues
  — workshop CR upload to EasyChair is the actual deadline.
- **The hyperref warning** ("Submission v1.5 does not automatically load
  hyperref.sty") is informational. Most class files (acmart, IEEEtran,
  llncs) load hyperref themselves. Don't add a duplicate `\usepackage`
  in your preamble or you'll get an option-clash error.
- **The "ACM reference format is mandatory" warning** is fine for arXiv;
  it only matters for the eventual ACM-published version, which goes
  through a separate e-rights cycle.

## Common arXiv compile gotchas

| Symptom | Cause | Fix |
|---|---|---|
| BibTeX errors but you shipped `.bbl` | Probably stale `.bbl` from a different version | Rebuild `.bbl` from current `references.bib`; ship the fresh one |
| "File not found" warnings on `acmart` images | acmart references CC-license / journal logos statically | Informational; the PDF still renders if `printacmref=false` is set |
| "Token not allowed in PDF string" hyperref warnings | Special chars (kerning, italics) in author names | Cosmetic; hyperref strips formatting from PDF metadata strings |
| ORCID iD invisible | acmart's `\orcid{}` only links the name, no visible icon | Load `\usepackage{orcidlink}` and use `\orcidlink{<id>}` next to author name; remove `\orcid{}` if you also want a visible href to the URL |

## Verify hyperlinks resolve

```bash
pip install --break-system-packages --user pypdf -q
python3 <<'EOF'
from pypdf import PdfReader
r = PdfReader('paper/main.pdf')
for i, page in enumerate(r.pages):
    if '/Annots' in page:
        for a in page['/Annots']:
            obj = a.get_object()
            if obj.get('/Subtype') == '/Link' and '/A' in obj:
                action = obj['/A'].get_object()
                if '/URI' in action:
                    print(f'page {i+1}: {action["/URI"]}')
EOF
```

This catches "linked but invisible" issues (the canonical ORCID bug) before
arXiv reviewers see them.
